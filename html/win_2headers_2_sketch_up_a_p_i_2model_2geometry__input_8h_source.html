21 /**
22 @brief SUMaterialInput contains information that is needed to apply a
23  material to a face. The conventional method for applying a material to
24  a face is to use 1 to 4 UV coordinates, which are Cartesian textures
25  coordinates and corresponding vertex indices on the face. The vertices
26  are referenced by index into the top level SUGeometryInputRef's vertex
27  array. The member variable 'material' must be a valid material object,
28  and once the material input is used (e.g. with /ref SUEntitiesFill),
29  the material object must not be released since it will be associated
30  with a parent object.
31 */
32 struct SUMaterialInput {
33  size_t num_uv_coords; ///< Number of texture coordinates. 0 if no texture.
34  ///< 1 to 4 otherwise.
35 
36  struct SUPoint2D uv_coords[4]; ///< Texture coordinates.
37 
38  size_t vertex_indices[4]; ///< Vertex indices corresponding to the texture
39  ///< coordinates. Should reference the vertex array
40  ///< of the parent SUGeometryInputRef.
41 
42  SUMaterialRef material; ///< Material to be applied.
43 };

282 /**
283 @brief Adds a vertex index to a loop input object.
284 @warning *** Breaking Change: The behavior of this method was changed in
285  SketchUp 2017 M2, API 5.2. In previous releases this method returned
286  \ref SU_ERROR_INVALID_INPUT if the specified index was already anywhere
287  in the loop. In SketchUp 2017 M1 the concept of an explicitly closed
288  loop was introduced. A loop can be explicitly closed by either using
289  this method to insert an index which is already at the beginning of the
290  loop, or by adding a curve to the loop which connects the loop's start

297  not at the front \ref SU_ERROR_INVALID_ARGUMENT will be returned.
298 @param[in] loop_input The loop input object.
299 @param[in] vertex_index The vertex index to add. This references a vertex within
300  the parent geometry input's vertex collection (as a
301  zero- based index).
302 @return
303 - \ref SU_ERROR_NONE on success
304 - \ref SU_ERROR_INVALID_INPUT if loop_input is not valid
305 - \ref SU_ERROR_UNSUPPORTED if the loop was already closed
306 - \ref SU_ERROR_INVALID_ARGUMENT if vertex_index already existed in the loop not

SUMaterialInput
SUMaterialInput contains information that is needed to apply a material to a face. The conventional method for applying a material to a face is to use 1 to 4 UV coordinates, which are Cartesian textures coordinates and corresponding vertex indices on the face. The vertices are referenced by index into the top level SUGeometryInputRef's vertex array. The member variable 'material' must be a valid material object, and once the material input is used (e.g. with /ref SUEntitiesFill), the material object must not be released since it will be associated with a parent object.
