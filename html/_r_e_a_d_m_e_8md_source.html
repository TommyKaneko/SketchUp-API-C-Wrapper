3 ================
4 ## Introduction
5 
6 SketchUp C API is made for the C programming language:
7 [ml]: http://extensions.sketchup.com/developer_center/sketchup_c_api/sketchup/index.html
8 
9 For C++ programmers, the non-object-orientated nature of the C API means simple operations for retrieving objects through is time consuming. This project provides a wrapper class for every SketchUp C API object that considerably shortens the amount of code required to perform simple tasks.
10 
11 For example, to get an array of faces from an entities object, you could write this:
12 ```
13 SUEntitiesRef entities = SU_INVALID;
14 SUModelGetEntities(model, &entities);
15 size_t faceCount = 0;
16 SUEntitiesGetNumFaces(entities, &faceCount);
17 std::vector<SUFaceRef> faces[faceCount];
18 SUEntitiesGetFaces(entities, faceCount, &faces[0], &faceCount);
19 ```
20 With the C++ Wrapper, to achieve the above, you just need to write this:
21 ```
22 #include <SUAPI-CppWrapper/model/Model.hpp>
23 #include <SUAPI-CppWrapper/model/Face.hpp>
24 std::vector<CW::Face> faces = CW::Model(model).entities().faces();
25 ```
26 Added to this benefit, the wrapper classes have helper functions to help with geometric manipulations, much in the style of the SketchUp Ruby API

80 ## Project Objectives
81 
82 The aims of this set of wrapper classes are:
83 - to make the SketchUp C API quicker and easier to use, from a C++ programmer's perspective.
84 - to provide all methods in the SketchUp C API, but in a simplified form.
85 - to provide all methods that are given in the equivalent classes in the SketchUp Ruby API ( [ml]: http://www.sketchup.com/intl/en/developer/docs/index ), if it is possible to deliver through the SketchUp C API.

88 ==========================
89 ## Programming Principles
90 
91 ##### Data Structure
92 Each C++ Wrapper class object holds its equivalent C API class object as a member variable (ie - CW::Face holds SUFaceRef). Further variables should be kept to a minimum, if at all.
93 
94 ##### Releasing Objects
95 With the use of the C++ Wrapper, the SketchUp API should "feel" like a native C++ API. This means the idea of "releasing" objects that are not attached to a model object, for example, should not have to be considered by the programmer. Instead, the wrapper class will deal with releasing of an object when the object goes out of scope.
96 
97 The only exception to this rule is if the wrapped object (`CW::Face`, for example), is converted to a regular C object ( `SUFaceRef`, for example), in which case it can be assumed that the programmer is taking charge of the C object and will be dealing with any releasing.
98 
99 ##### Method Names
100 Method names take inspiration mostly from the SketchUp Ruby API. That said, these are not wholy consistent, and some liberties have been taken to make the method names make sense.
